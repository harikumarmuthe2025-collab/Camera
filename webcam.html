Enterprise live camera attendance.

This page publishes a continuous high-quality WebRTC camera stream for face attendance. It requests 1920×1080 video, targets 30 FPS, limits sender bitrate to 4.5 Mbps, and sends only signaling plus secure attendance metadata to your backend.

This is true WebRTC livestreaming. It does not use MediaRecorder and does not upload video blobs. Mobile camera access is requested safely first, then upgraded to the best available high-quality face-detection video. PHP can manage sessions, users, metadata, and attendance records, but a WebRTC receiver/SFU/media server should handle the real-time video path.
